# АстроПульс — Telegram-бот персональных астропрогнозов

[![CI](https://github.com/r3lax3/astrobot/actions/workflows/ci.yml/badge.svg)](https://github.com/r3lax3/astrobot/actions/workflows/ci.yml)
![Python 3.11+](https://img.shields.io/badge/python-3.11+-blue)
![aiogram 3](https://img.shields.io/badge/aiogram-3.x-2CA5E0)

Продакшн-бот, который считает персональные астрологические прогнозы по натальной карте
пользователя и присылает их каждый день в выбранное время — с учётом часового пояса
и координат. Расчёты делаются локально через швейцарские эфемериды, без сторонних
астро-API.

## Возможности

- **Персональный ежедневный прогноз** — транзитные аспекты к натальной карте
  (Swiss Ephemeris), текстовые толкования + сгенерированная картинка с астроданными.
- **Лунный календарь** — лунные дни, фазы, Луна в знаке, периоды «холостой» Луны.
- **Подборка благоприятных дней** и толкование снов по лунным дням.
- **Платная подписка** — оплата через Prodamus (ссылки с HMAC-подписью,
  верификация входящих вебхуков), промокоды, напоминания о продлении.
- **Админ-панель** — рассылки, статистика, общие прогнозы, карта дня,
  автоматический бэкап базы в админ-чат.

## Стек

| Слой | Технологии |
|---|---|
| Бот | Python 3.11, aiogram 3 (webhook-режим), aiohttp |
| Состояния | Redis (FSM-хранилище) |
| База | SQLite + SQLAlchemy 2 |
| Планировщик | APScheduler — персональные рассылки по cron с таймзоной пользователя |
| Астрорасчёты | pyswisseph (Swiss Ephemeris), ephem |
| Изображения | Pillow — генерация карточек прогноза |
| Качество | pytest, ruff, GitHub Actions |

## Архитектура

```
main.py                  # сборка приложения: aiohttp-сервер, вебхуки, middleware
src/
├── astro_engine/        # чистые астрорасчёты (аспекты, лунные дни, фазы, знаки)
├── routers/
│   ├── user/            # пользовательские сценарии (прогнозы, подписка, профиль)
│   └── admin/           # админ-панель (рассылки, статистика, контент)
├── payments/            # Prodamus: генерация подписанных ссылок, вебхуки оплаты
├── scheduler.py         # ежедневные рассылки и напоминания о продлении
├── image_processing/    # генерация картинок с астроданными
├── database/            # модели, CRUD, бэкапы
├── middlewares/         # очистка клавиатур, медиагруппы, фильтрация апдейтов
└── keyboards/           # декларативное описание клавиатур
data/                    # контент: толкования, сны, подборка дней (CSV/JSON)
docs/architecture.drawio # схема проекта
```

Схема взаимодействия компонентов: [docs/architecture.drawio](docs/architecture.drawio)
(открывается на [diagrams.net](https://app.diagrams.net)).

> **Почему контент в CSV, а не в БД?** Тексты толкований правит астролог —
> вручную, в таблицах, без админки. CSV-файлы — источник истины: их можно
> ревьюить в git и заменять без миграций. Осознанный трейдофф в пользу
> простоты процесса.

## Запуск

Понадобятся: Python 3.11+, Redis, домен с HTTPS для вебхука Telegram.

```bash
git clone https://github.com/r3lax3/astrobot.git && cd astrobot
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env                    # токен бота
cp config.example.toml config.toml      # вебхук, платежи, админы
python main.py
```

## Тесты и линт

```bash
pip install -r requirements-dev.txt
cp config.example.toml config.toml
pytest        # юнит-тесты: астрорасчёты, подписи платежей, генерация изображений
ruff check .  # линт
```

Астрорасчёты покрыты тестами с эталонными значениями из астрологических
справочников (точность до минут). Интеграционные тесты, требующие внешних
сервисов, включаются переменной `RUN_INTEGRATION_TESTS=1`.
